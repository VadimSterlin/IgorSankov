## 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

**Цель:** Создать лаконичный одностраничный сайт-визитку благотворительного характера для сбора средств на реабилитацию Игоря Санкова (28 лет, диагноз: пояснично-конечностная мышечная дистрофия, тип R9C5).

**Целевая аудитория:** Англоязычная и русскоязычная аудитория Twitter/X и других соцсетей.

**Ключевые требования:**
- Двуязычность (RU / EN) с переключателем языка
- Адаптивная вёрстка (mobile-first, т.к. основной трафик — из Twitter/X)
- Быстрая загрузка (минимум зависимостей)
- Корректное отображение при шаринге в Twitter/X (OpenGraph / Twitter Card мета-теги)
- Реквизиты (крипто-адреса) легко копируются в один клик
- Документы с заблюренными персональными данными (номера паспортов)
- Визуальный стиль: чистый, сдержанный, вызывающий доверие (как у благотворительных фондов)

## 2. ТЕХНИЧЕСКИЙ СТЕК

| Компонент | Решение | Обоснование |
|---|---|---|
| Фреймворк | Чистый HTML + CSS + минимальный JS | Максимальная скорость, нет зависимостей, легко хостить на любом статик-хостинге |
| Языки | i18n через JS-объект с переводами | Простая реализация без фреймворков |

| Иконки | Inline SVG для соцсетей и крипто | Без внешних зависимостей |
| Хостинг | GitHub Pages / Netlify / Vercel (не .ru домен) | Требование заказчика — не российский сервер |

### 7.4. Smooth scroll
- Навигационные ссылки плавно скроллят к секциям
- `scroll-behavior: smooth` + JS fallback

### 7.5. Кнопка «Share on X»
- Формирует URL: `https://twitter.com/intent/tweet?text=...&url=...`
- Предзаполненный текст: "Help Igor Sankov fight muscular dystrophy. Every share matters. 🤝"

## Выявленные проблемы (Ревью №1)

**Описание:** Для крипто-платежей пользователям удобнее сканировать QR-код с мобильного кошелька, чем копировать длинный адрес.

**Описание:** Благотворительные сайты обычно показывают прогресс сбора. Однако, для крипто-кошельков автоматическое отслеживание сложно, и заказчик не просил этого.
**Решение:** Опционально — добавить ручной прогресс-бар, который Игорь может обновлять, меняя одно число в HTML. Или не добавлять, чтобы не усложнять.

### Проблема 3: Favicon не определён
**Описание:** Нет исходника для favicon.
**Решение:** Создать простой favicon — инициалы «IS» или символ сердца/руки помощи. Сгенерировать через CSS/SVG.

### Проблема 4: Нет alt-текстов для изображений

**Решение:** Добавить alt-тексты для каждого изображения на обоих языках.

### Проблема 5: Не указана обработка ошибок для clipboard API
**Описание:** navigator.clipboard может быть недоступен в некоторых браузерах/контекстах (HTTP, старые браузеры).

## Выявленные проблемы (Ревью №2)

### Проблема 1: Вес видео для мобильного трафика
**Описание:** Даже после сжатия видео могут весить 5-10 МБ суммарно, что замедлит загрузку на мобильном.

### Проблема 2: Google Fonts — внешняя зависимость
**Описание:** Если Google Fonts недоступен (блокировка), шрифты не загрузятся.
**Решение:** Скачать шрифты локально и подключить через @font-face. Добавить в файловую структуру папку `fonts/`.

**Описание:** Указана только кнопка Share для Twitter/X. Но аудитория может быть и в Telegram, WhatsApp.
**Решение:** Добавить кнопки шаринга: Twitter/X, Telegram, WhatsApp, Copy Link. Реализовать через нативные URL-схемы (не внешние виджеты).

### Проблема 4: Нет email для официальных обращений
